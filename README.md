# 途悦

本项目“途悦（Tuyue）”定位为一个基于 B/S 架构的综合性旅游资讯与互动平台，强调“内容即核心资产”的设计理念。系统以文章与视频作为主要内容载体，通过首页推荐流、文章列表、视频列表等页面为用户提供结构化的旅游信息获取入口；同时设置讨论区模块承载用户生成内容（UGC），用户可以在帖子详情页进行点赞、收藏、关注作者以及评论/回复等互动，从而形成“内容消费—互动沉淀—再传播”的社区闭环。

本项目已开源，开源版本为本地运行版本 https://github.com/Chunran7/Database

## 2. 技术栈说明

- **后端 (Back-end)**：Spring Boot 3.2.0, MyBatis, MySQL 8.x, JWT (安全认证), MD5 (密码加密), Maven。
- **前端 (Front-end)**：Vue 3 (Composition API), Vite, Element Plus (UI 库), Axios (请求封装), Vue Router。

## 3. 核心功能模块

### 3.1 门户功能

- **用户模块**：支持注册、登录、个人中心编辑及用户关注体系。
- **内容模块**：支持文章发布、社区发帖、视频上传与在线流媒体播放。
- **互动模块**：实现点赞、收藏、浏览量统计及多级评论（支持根评论与二级回复）。

### 3.2 管理员后台

- **全站统计**：可视化展示用户、文章、视频及帖子的实时数据总量。
- **内容审计**：支持对违规内容进行逻辑删除（下架）、物理硬删除或一键恢复。
- **用户准入**：监控用户状态，支持对违规账号进行实时封禁或解封。
- **安全防御**：通过后台专用拦截器 `AdminLoginInterceptor` 配合 JWT 进行身份校验。

## 4. 数据库实现细节

- **级联一致性 (Referential Integrity)**：在 `post_like`、`comment` 等表中使用 `ON DELETE CASCADE` 约束。当管理员物理删除主文章或帖子时，其关联的数千条点赞及回复记录将由数据库引擎自动清理，确保数据无冗余。
- **逻辑删除机制 (Soft Delete)**：核心业务表（`article`, `post`, `video`）均设有 `is_deleted` 标记（0:未删, 1:已删），管理员执行“删除”时仅更新状态，不破坏业务历史数据链路。
- **动态 SQL 筛选**：利用 MyBatis 的 `<if>` 和 `<where>` 标签实现多维度模糊搜索，配合物理分页（`LIMIT/OFFSET`）大幅降低单次数据库 I/O 开销。
- **权限字段扩展**：用户表 `user` 新增了 `status` 状态位，实现了灵活的全局权限开关逻辑。

## 5. 本地运行指南 (详细步骤)

### 5.1 环境需求

- **JDK**: 17 或更高版本。
- **Node.js**: 16.x 或更高版本。
- **MySQL**: 8.0 或更高版本。

### 5.2 步骤一：数据库初始化

1. 登录 MySQL 管理工具（如 Navicat 或命令行）。
2. 创建数据库：`CREATE DATABASE IF NOT EXISTS dbproject DEFAULT CHARACTER SET utf8mb4;`。
3. **运行脚本**：依次导入并执行项目中的 SQL 文件：
   - 执行 `code/back/src/main/resources/schema.sql` (创建表结构)。
   - 执行 `code/back/src/main/resources/test_data_generated.sql` (导入初始测试数据与管理员账号)。

### 5.3 步骤二：后端项目配置与运行

1. 进入后端目录：`cd code/back`。
2. **配置数据库**：编辑 `src/main/resources/application.yml` 文件，将 `username` 和 `password` 修改为您本地的 MySQL 账号密码。
3. **安装依赖**：使用 Maven 加载依赖项。
4. **启动项目**：运行 `src/main/java/com/chun/back/BackApplication.java` 中的 `main` 方法。
   - 后端默认运行在：`http://localhost:8080`。

### 5.4 步骤三：前端项目配置与运行

1. 进入前端目录：`cd code/front`。
2. **安装依赖**：运行 `npm install`。
3. **启动开发服务器**：运行 `npm run dev`。
   - 访问地址：`http://localhost:5173` (具体以终端输出为准)。

### 5.5 步骤四：功能验证与登录

- **普通用户**：使用测试数据中的用户账号（如 `admin` / `admin123`）登录。
- **管理员后台**：访问 `/admin` 路由（或从导航栏进入），使用初始管理员账号进行登录测试。
